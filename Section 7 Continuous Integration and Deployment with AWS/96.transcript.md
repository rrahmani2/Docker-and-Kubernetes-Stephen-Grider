Here’s a clear breakdown of the last section on **setting up AWS credentials for Travis CI deployment**:

---

### **Giving Travis CI Access to AWS**

**Goal:** Allow Travis CI to deploy your app to Elastic Beanstalk securely.

---

### **Steps**

1. **Create a new IAM user in AWS:**

   * Go to **AWS Console → Services → IAM → Users → Add User**.
   * Name it something descriptive (e.g., `DockerReactTravisCI`).
   * **Access type:** Programmatic access only (for API calls, not console login).

2. **Assign permissions:**

   * Click **Attach existing policies directly**.
   * Search for `Elastic Beanstalk`.
   * Select **AWS Elastic Beanstalk Full Access**.
   * Complete the user creation.

3. **Save the API keys:**

   * You will get:

     * **Access Key ID**
     * **Secret Access Key**
   * **Important:** The secret key is only shown once. Save it securely.

4. **Store keys securely in Travis CI:**

   * Go to **Travis CI Dashboard → Your Project → More Options → Settings → Environment Variables**.
   * Add two variables:

     * `AWS_ACCESS_KEY` → paste Access Key ID
     * `AWS_SECRET_KEY` → paste Secret Access Key
   * Make sure **“Display value in build log”** is **unchecked**.

5. **Reference keys in `.travis.yml`:**

   ```yaml
   access_key_id: "$AWS_ACCESS_KEY"
   secret_access_key: "$AWS_SECRET_KEY"
   ```

   * Wrap in quotes if needed:

     ```yaml
     access_key_id: "$AWS_ACCESS_KEY"
     secret_access_key: "$AWS_SECRET_KEY"
     ```

6. **Commit and push the updated Travis configuration:**

   ```bash
   git add .travis.yml
   git commit -m "Added Travis deploy config"
   git push origin master
   ```

---

### **Outcome**

* Travis CI can now deploy your application automatically to Elastic Beanstalk **after tests pass**, without exposing your AWS credentials publicly.
* Pushing to master triggers the full CI/CD flow:
  **GitHub push → Travis builds/tests → Travis deploys to AWS**.
