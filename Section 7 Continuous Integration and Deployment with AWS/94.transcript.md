### **Automating Deployment with Travis CI**

**Goal:** Deploy the application to AWS automatically after tests pass.

**Steps in `.travis.yml`:**

1. **Add a `deploy` section** to the file.

   ```yaml
   deploy:
     provider: elasticbeanstalk
   ```

2. **Specify the AWS region** where your Elastic Beanstalk environment resides.

   ```yaml
   region: "us-west-2"  # replace with your region
   ```

3. **Provide application and environment names:**

   ```yaml
   app: "docker"          # your Elastic Beanstalk app name
   env: "docker-env"      # your Elastic Beanstalk environment name
   ```

4. **Set up the S3 bucket and path:**

   * **Bucket:** AWS automatically creates a bucket for Elastic Beanstalk; copy the name from S3.
   * **Bucket path:** Folder name inside the bucket, usually the app name.

   ```yaml
   bucket_name: "elasticbeanstalk-us-west-2-xxxxxx"
   bucket_path: "docker"  # matches your app name
   ```

5. **Deploy only from the master branch:**
   Prevents automatic deployment from feature branches.

   ```yaml
   on:
     branch: master
   ```

---

**Key Concepts:**

* Travis CI will zip your repository, upload it to the S3 bucket, and tell Elastic Beanstalk to deploy from that zip.
* Deployment only triggers on the **master branch**, keeping feature branch work isolated.
* This sets up a **continuous deployment workflow**: tests pass → code deploys automatically.
