### Cert-Manager Setup — Technical Notes

**Objective:**
Enable automated TLS certificate management in Kubernetes through integration with **Let’s Encrypt** using **Cert-Manager**.

---

#### 1. **Purpose of Cert-Manager**

* Automates:

  * **Requesting**, **issuing**, **renewing**, and **storing** TLS certificates.
  * Certificate management for Kubernetes Ingress resources.
* Integrates directly with **Let’s Encrypt** or other certificate authorities.
* Eliminates manual certificate creation or renewal processes.

---

#### 2. **Installation Source**

**Repository:** [https://github.com/jetstack/cert-manager](https://github.com/jetstack/cert-manager)

**Documentation Path:**
→ *Documentation* → *Getting Started* → *Installing Cert-Manager with Helm*

---

#### 3. **Installation Commands**

In Cloud Shell (or any terminal with `kubectl` and `helm` access to the cluster):

```bash
helm repo add jetstack https://charts.jetstack.io
helm repo update

helm install cert-manager jetstack/cert-manager \
  --namespace cert-manager \
  --create-namespace \
  --set installCRDs=true
```

**Explanation:**

* `--namespace cert-manager`: Creates a dedicated namespace for Cert-Manager components.
* `--set installCRDs=true`: Ensures Custom Resource Definitions (CRDs) required by Cert-Manager are installed automatically.

---

#### 4. **What Gets Deployed**

Helm creates several Kubernetes resources:

* **Pods** for Cert-Manager controllers and webhooks
* **Deployments** managing Cert-Manager’s lifecycle
* **ClusterRoles** and **RoleBindings** granting necessary permissions
* **ServiceAccounts** for secure communication
* **Services** to expose internal APIs

---

#### 5. **Validation**

After installation, confirm all components are running:

```bash
kubectl get pods -n cert-manager
```

Expected output:

```
NAME                                      READY   STATUS    RESTARTS   AGE
cert-manager-xxxxxx                       1/1     Running   0          1m
cert-manager-cainjector-xxxxxx            1/1     Running   0          1m
cert-manager-webhook-xxxxxx               1/1     Running   0          1m
```

---

#### 6. **Next Step**

* Understand how Cert-Manager automates certificate issuance through **ClusterIssuer** or **Issuer** resources.
* Configure integration with **Let’s Encrypt** to issue and renew TLS certificates automatically for your domain.

---

**Summary:**
Cert-Manager is installed into the Kubernetes cluster via Helm to automate the lifecycle of TLS certificates. It establishes the base infrastructure required for communication with Let’s Encrypt, enabling secure HTTPS support through automated certificate management.
