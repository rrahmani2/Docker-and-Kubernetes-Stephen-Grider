### **Development vs Production in React**

1. **Development Environment**

   * Uses a **development server** (`npm run start`).
   * Dev server dynamically processes and serves your JS files and `index.html`.
   * Responds to browser requests on port 3000.
   * Necessary because code is frequently changing, and React hot-reloads changes.

2. **Production Environment**

   * No development server.
   * You run `npm run build` **once**, which generates static files:

     * `index.html`
     * A bundled JavaScript file containing all React code.
   * These files are served to users **without any processing**.
   * Running a dev server in production is unnecessary and inefficient.

---

### **Solution: Use NGINX**

* NGINX is a lightweight, high-performance web server.
* In production, it serves static files (the `index.html` and JS bundle).
* Steps for production container:

  1. Create a **new Dockerfile** for production.
  2. Install NGINX inside the container.
  3. Copy the built React files into NGINX’s serving directory.
  4. NGINX serves the app to incoming browser requests.
