### **Testing the Production Dockerfile**

1. **Build the image**
   Since the Dockerfile follows the standard naming convention (`Dockerfile`), you can simply run:

   ```bash
   docker build .
   ```

   * Docker executes the multi-stage build.
   * Stage 1 (`builder`): runs `npm install` and `npm run build`.
   * Stage 2: uses the NGINX image and copies `/app/build` into `/usr/share/nginx/html`.

2. **Run the container**
   Map host port `8080` to NGINX’s default port `80`:

   ```bash
   docker run -p 8080:80 <image_id>
   ```

   * `<image_id>` is the ID returned from the build step.

3. **Test in browser**

   * Open: `http://localhost:8080`
   * Your React app should appear, served by NGINX.

---

### **Key Takeaways**

* Multi-stage build ensures **small production image**: only NGINX + built assets.
* NGINX is **production-ready**, unlike the dev server.
* No need for Node, volumes, or dev dependencies in production.
