### **Setting Up a Development Dockerfile**

We’re creating a Dockerfile specifically for development. Here’s how it works:

---

#### **Step 1: Create the Dev Dockerfile**

Inside your project folder:

```bash
touch Dockerfile.dev
```

---

#### **Step 2: Dockerfile.dev Contents**

```dockerfile
# Use Node.js base image
FROM node:alpine

# Set working directory inside container
WORKDIR /app

# Copy package.json and install dependencies
COPY package.json .
RUN npm install

# Copy the rest of the project files
COPY . .

# Start the development server
CMD ["npm", "run", "start"]
```

**Notes:**

* `Dockerfile.dev` is only for development.
* We will create a separate `Dockerfile` (no suffix) for production.
* This setup installs dependencies and starts the dev server (`npm run start`).

---

#### **Step 3: Build the Image Using the Dev Dockerfile**

By default, `docker build` looks for a file named `Dockerfile`. Since we named ours `Dockerfile.dev`, we need to specify it:

```bash
docker build -f Dockerfile.dev -t react-dev-image .
```

**Explanation:**

* `-f Dockerfile.dev` → specifies the Dockerfile to use.
* `-t react-dev-image` → gives the image a name (`react-dev-image`).
* `.` → current directory as build context.

---

#### **Step 4: Run the Container (Optional)**

Once the image is built:

```bash
docker run -p 3000:3000 react-dev-image
```

* Maps port `3000` on your host to port `3000` in the container.
* Allows you to view your React app in the browser at `localhost:3000`.

---

**Summary:**

* Use `Dockerfile.dev` for development builds.
* Build the image with `-f` to specify a non-standard Dockerfile name.
* Production will use a separate `Dockerfile`.
