### **Volume Mounting Recap**

* You now have **two volumes** in your development container:

  1. `app/node_modules` → preserves container-installed dependencies.
  2. `./` (current folder) → maps all your local source code into the container (`/app`).

* This allows **hot-reloading**: any changes to local files automatically reflect in the running container.

---

### **About the COPY step in Dockerfile.dev**

* Technically, the line that copies your source code into the container is now **redundant** because the volume mount overrides it.
* However, **keep it** for two reasons:

  1. Future-proofing: if you stop using Docker Compose, you’ll still need this for the container to have the source code.
  2. Reference for production: when creating a production Dockerfile, you’ll need to copy the source code into the image.

---

* Your React app runs inside Docker with live updates.
* Development workflow is simplified and safe for experimentation.
* Dockerfile.dev can serve as a template for production adjustments later.
