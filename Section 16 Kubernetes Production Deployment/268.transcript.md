### Travis CI — Google Cloud SDK Configuration

1. **Verify Encrypted File**

   * Inside the project folder (`complex`), ensure that **only** the encrypted service account file exists:

     ```
     service-account.json.enc
     ```
   * The original unencrypted file must be permanently deleted.

2. **Decrypt File in Travis**

   * The `.travis.yml` file already includes the OpenSSL command (added earlier) in the `before_install` section:

     ```yaml
     - openssl aes-256-cbc -K $encrypted_<key>_key -iv $encrypted_<key>_iv \
       -in service-account.json.enc -out service-account.json -d
     ```
   * This decrypts the service account file during the build and places it in the project root.
   * The decrypted file is then used by the Google Cloud SDK for authentication.

3. **Authenticate Google Cloud SDK**

   * Add the following command to activate the service account using the decrypted JSON key:

     ```yaml
     - gcloud auth activate-service-account --key-file service-account.json
     ```

4. **Configure Google Cloud Project and Zone**

   * Each Google Cloud account can have multiple projects, so specify which project Travis should use.

   * Use the **project ID**, not the display name (visible in the project selector’s right column):

     ```yaml
     - gcloud config set project <project-id>
     ```

     Example:

     ```yaml
     - gcloud config set project skillful-berm-123456
     ```

   * Set the **default compute zone** (found under *Kubernetes Engine → Clusters → Location*):

     ```yaml
     - gcloud config set compute/zone us-central1-a
     ```

5. **Configure Target Cluster**

   * Specify the cluster for Kubernetes commands:

     ```yaml
     - gcloud container clusters get-credentials multi-cluster
     ```
   * This command authenticates the current session with the specified Kubernetes cluster and updates the local kubeconfig.

6. **Summary of Configuration Section in `.travis.yml`**

   ```yaml
   before_install:
     - openssl aes-256-cbc -K $encrypted_<key>_key -iv $encrypted_<key>_iv \
       -in service-account.json.enc -out service-account.json -d
     - gcloud auth activate-service-account --key-file service-account.json
     - gcloud config set project skillful-berm-123456
     - gcloud config set compute/zone us-central1-a
     - gcloud container clusters get-credentials multi-cluster
   ```

7. **Next Steps**

   * After SDK and cluster configuration:

     * Log into Docker CLI.
     * Build the test version of the image.
     * Run test suites before proceeding with deployment.
