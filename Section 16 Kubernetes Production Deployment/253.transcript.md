### Step 1: Create a GitHub Repository for the Kubernetes Project

1. **Create a new repository**

   * Go to GitHub → click the **“+”** icon (top right) → select **New repository**.
   * Name it something like `multi-k8s` (or any preferred name).
   * Set visibility to **Public**, then click **Create Repository**.
   * Copy the repository’s HTTPS/SSH link.

2. **Initialize or update your local Git repo**

   * Navigate to your local project directory (e.g., `complex/`).

   * Run:

     ```bash
     ls -a
     ```

     If you see a `.git` folder, a local repository already exists — skip the `git init` step.

   * If no `.git` folder exists, create one and commit:

     ```bash
     git init
     git add .
     git commit -m "Initial commit for Kubernetes setup"
     ```

3. **Update Git remote (if necessary)**

   * Check existing remotes:

     ```bash
     git remote -v
     ```
   * If a remote named `origin` already points to the old Elastic Beanstalk repo, remove it:

     ```bash
     git remote remove origin
     ```
   * Add the new remote:

     ```bash
     git remote add origin <your-new-repo-URL>
     ```
   * Verify:

     ```bash
     git remote -v
     ```

4. **Push local code to GitHub**

   ```bash
   git push origin master
   ```

   Wait for the push to complete.

5. **Verify repository contents**

   * Refresh your GitHub page.
   * Ensure it contains:

     * `client/`, `server/`, and `worker/` directories.
     * A `k8s/` (or `k8s-config/`) folder with all Kubernetes YAML files.
   * Optional cleanup: remove macOS `.DS_Store` files if present.
